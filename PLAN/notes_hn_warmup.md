- [ ] Refresh profile *now*: real name, URL = `https://amiheines.com`, 1-line bio (e.g. "Self-hosted infra + on-chain games"). Empty profile = burner-account signal.
- [ ] Read HN front page + `/newest` daily. ~20 min/day. **No posts, no submissions, no comments yet.**
- [ ] Build context: track which threads in your niches (`Antelope` / `EOS` / `self-hosted` / `bootstrapped` / `IPFS` / `web3`) get upvoted vs flagged. What patterns? What opens?
- [ ] Note 5 commenters whose voice you respect — that's the register to aim for.

## Phase 2 — Substantive commenting (W03–W05, May 18 → Jun 7)

- [ ] **5–10 comments/week.** Quality over quantity. Aim 200–500 words each.
- [ ] Add facts / links to source / personal experience / counterpoints. Never "+1" or generic agreement.
- [ ] Comment only on threads where you have real expertise — Antelope ops, self-hosting, smart-contract patterns, indie hacking, chain economics.
- [ ] **No links to your own domain in comments.** Auto-mod flags single-purpose accounts hard.
- [ ] **Avoid:** marketing speak, ChatGPT-sounding text (HN is hyper-allergic), defensive replies to disagreement, flame-war engagement.
- [ ] Karma target by end of W05: **≥50**. Earns flagging power + signals you're a real voice.

## Phase 3 — Submit OTHERS' content (W06–W07, Jun 8–21)

- [ ] Submit 2–3 genuinely interesting articles from **other people's blogs** in your niche.
- [ ] Builds reputation as "this person finds good stuff" — not "this person promotes themselves."
- [ ] Optimal window: **Tuesday/Wednesday 8–10am US Eastern** (~15:00–17:00 IST). Avoid weekends + US holidays.
- [ ] Track which submissions land vs flop — calibrate before W08 self-shot.

## Phase 4 — First self-submission (W08, Jun 22–28)

- [ ] Title format: `Show HN: [thing] – [one-line value prop, no hype]`. Example: `Show HN: I run an Antelope chain on €300/mo of OVH – here's the breakdown`.
- [ ] First comment under your submission = the technical "actual novel thing" + how it differs from existing alternatives. **No CTAs, no "subscribe to my newsletter."**
- [ ] **Never ask for upvotes** — not on Twitter, LinkedIn, Slack, Discord, anywhere. Detected = instant kill + permanent account-reputation damage.
- [ ] Engage technical questions in comments. Ignore vibes / hot takes — don't get drawn into defending the post.
- [ ] If post tanks unfairly within 1–2h: email `hn@ycombinator.com` with a calm, factual case. dang the moderator is human and replies. Only email if you actually have a case.

## Ongoing cadence (W09+)

- **Self-submissions:** ≤1/month for own URLs. Anything more reads as promotional and tanks account reputation.
- **Others' submissions:** 5+/month. Keeps you a contributor, not a marketer.
- **Comments:** continue 5–10/week minimum. The comment-to-submission ratio is what HN's reputation system rewards.

## Things that get accounts shadow-banned silently

- Submitting your own URL within 24h of cross-posting it on Twitter/LinkedIn (the cross-post timing is fingerprinted).
- Multiple accounts from the same IP voting on your own post (don't ask family/friends to upvote either — they're often on the same network).
- Defensive replies to every comment under your submission. Just answer technical Qs, ignore the rest.
- Promotional language anywhere — body, replies, profile bio.
- Generic / AI-sounding comments. Repeat: HN is hyper-allergic.
